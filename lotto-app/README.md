# ตรวจหวย — เว็บผลสลากกินแบ่งรัฐบาล + สถิติ + แจ้งเตือน

เว็บตรวจหวยที่ออกแบบให้ **เกาะฐานคนดูไลฟ์ของคุณ**: รายงานผลสดระหว่างถ่ายทอด (กรอกเองทันทีที่หวยออก), ตรวจสลากหลายใบ, สถิติย้อนหลัง (หน้า SEO), และแจ้งเตือนผ่าน Telegram

Stack: Node.js (Express) + PostgreSQL + vanilla HTML/CSS/JS — โครงเดียวกับ LINER NOTES, deploy Railway ได้เลย

## โครงสร้าง

```
server.js        Express + API + กรอกผลสด + SSE
lib.js           ตรรกะหลัก: แปลงวันที่ไทย, ตรวจรางวัล, คำนวณสถิติ (มี unit test)
sources.js       ดึงผลจาก API ทางการ (GLO) + สำรอง (rayriffy) — ใช้ backfill/ตรวจทาน
telegram.js      บอทแจ้งเตือน (เปิดเมื่อใส่ TELEGRAM_BOT_TOKEN)
seed.js          โหลดข้อมูลย้อนหลัง 428 งวด (ปี 2549–2567) ลงฐานข้อมูล
seed_draws.json  ข้อมูลย้อนหลังจริง
public/          index.html (ตรวจหวย) · stats.html (สถิติ) · admin.html (กรอกผลสด)
```

## Deploy บน Railway

1. push โค้ดทั้งหมดไว้ที่ **root ของ repo** (ไม่ใส่ในโฟลเดอร์ย่อย — ตามบทเรียนเดิม) และ **ไม่ต้องมี `railway.json`**
2. เพิ่ม PostgreSQL ใน Railway → `DATABASE_URL` จะถูกตั้งให้อัตโนมัติ
3. ตั้ง Variables: `ADMIN_TOKEN` (รหัสยาว ๆ สุ่ม), และ `TELEGRAM_BOT_TOKEN` ถ้าจะใช้บอท
4. Deploy → Railway รัน `npm start`
5. seed ข้อมูลย้อนหลังครั้งเดียว: ใน Railway shell รัน `npm run seed`

## ใช้งานวันหวยออก (สำคัญ — จุดต่างของคุณ)

เปิด `/admin` ตอนไลฟ์ → ใส่ `ADMIN_TOKEN` → เลือกวันที่งวด แล้วกรอกผลทีละรางวัลตามที่ประกาศบนเวที กด **“บันทึก & รายงานสด”** ทุกครั้ง — ผู้ชมที่เปิดหน้าเว็บอยู่จะเห็นเลขเด้งขึ้นทันที (ผ่าน SSE) เร็วกว่าดึง API

พอครบทุกรางวัล กด **“ประกาศผลทางการ”** → ระบบส่งแจ้งเตือนให้ทุกคนที่เฝ้าเลขไว้ใน Telegram

> ปุ่ม **“ดึงจาก API ทางการ”** ไว้ใช้ตอนไม่ได้ไลฟ์ หรือเพื่อตรวจทานเลขที่กรอกเอง

## หมายเหตุเรื่องข้อมูล / กฎหมาย

ตัวเลขผลรางวัลเป็นข้อเท็จจริงสาธารณะ (ลิขสิทธิ์คุ้มครองไม่ได้) แต่ชุดข้อมูล Open Data ของ GLO ระบุสัญญาแบบ Non-Commercial — การ **กรอกผลเองจากไลฟ์ของคุณ** จึงเป็นแนวทางที่สะอาดที่สุดสำหรับเว็บที่มีโฆษณา ใช้ API ทางการเป็นแค่ตัวตรวจทาน

## แผนหารายได้ (ระยะถัดไป)

- **AdSense + แบนเนอร์ตรง** ช่วงพีค (วันที่ 1 และ 16) — ฝังลิงก์เว็บในไลฟ์/คอมเมนต์เพื่อดึงคนดูเข้าเว็บ
- **สมาชิกพรีเมียม**: เฝ้าเลขไม่จำกัด / ไม่มีโฆษณา / สถิติเชิงลึก
- ก่อนเปิดกว้าง: ระบบมี **rate limit ต่อ IP** อยู่แล้ว (60 req/นาที ทั่วไป, 30 req/นาทีสำหรับตรวจเลข) — ปรับได้ใน `server.js`

## v2 — หลายหวย + SEO

**หลายประเภทหวย (24 ประเภท / 5 หมวด):** รัฐบาล, ลาว, ฮานอย, หุ้นไทย, หุ้นต่างประเทศ — เลือกได้จากหน้าแรก แต่ละหวยแยกข้อมูลกันคนละชุด
- รัฐบาล = โครงสร้างเต็ม (รางวัลที่ 1–5, หน้า/ท้าย 3 ตัว, ท้าย 2 ตัว, เงินรางวัล)
- หวยอื่น = โครงสร้าง 3 ตัวบน + 2 ตัวล่าง (2 ตัวบนคำนวณให้อัตโนมัติ)
- กรอกผลทุกหวยได้ที่ `/admin` (ฟอร์มสลับตามประเภทหวยอัตโนมัติ)
- ยี่กี / ออมสิน / ธกส / มาเลย์ มีโครงสร้างพิเศษ ค่อยเพิ่มแบบเฉพาะทางภายหลัง

**หน้า SEO รายเลข (programmatic SEO):**
- `/huay/<เลข>` เช่น `/huay/23` (2 หลัก) หรือ `/huay/290` (3 หลัก) — render ฝั่งเซิร์ฟเวอร์ มี title/meta/canonical + ข้อมูลจริง (ออกกี่ครั้ง, ล่าสุดเมื่อไหร่, ประวัติย้อนหลัง, เลขใกล้เคียง)
- `/sitemap.xml` รวมทุกหน้า (00–99 + เลข 3 หลักที่เคยออก) และ `/robots.txt`
- **ตั้ง `BASE_URL`** ใน Variables เป็นโดเมนจริง เพื่อให้ canonical/sitemap ถูกต้อง
- ขั้นต่อไปด้าน SEO: เอา `/sitemap.xml` ไปส่งใน Google Search Console

**Migration:** initDb อัปเกรดตาราง `draws` เดิม (PK เป็น date) ให้เป็น (lottery, date) อัตโนมัติ ข้อมูลรัฐบาลเดิมไม่หาย

## Auto-sync หวยรัฐบาล (ไม่ต้องกรอกเอง)

- ดึงผลจาก GLO อัตโนมัติ: ตั้ง Variable `AUTO_SYNC=1` (และ `AUTO_SYNC_MINUTES=30` ปรับได้) ระบบจะดึงผลงวดล่าสุดเองเป็นระยะ อัปเดตหน้าเว็บ + ส่งแจ้งเตือน Telegram ให้อัตโนมัติ
- หรือกดดึงครั้งเดียวจากปุ่ม "ดึงผลรัฐบาลจาก API" ในหน้า `/admin`
- หวยลาว/ฮานอย/หุ้น ไม่มี API ฟรีทางการ — ใช้ API สำเร็จรูปแบบเสียเงิน (เช่น apilotto.com) แล้วเชื่อมเข้า `autoSync` ได้ภายหลัง การกรอกผลสดจากไลฟ์ยังเป็นจุดแข็งเฉพาะของหวยรัฐบาล

## v3 — landing รายหวย + โฆษณา + analytics + สถิติทุกหวย

- **หน้า landing รายหวย:** `/lotto/<code>` เช่น `/lotto/hanoi` = "ตรวจหวยฮานอยปกติวันนี้" (server-rendered, มี title/meta/canonical + ผลล่าสุด + ย้อนหลัง + ลิงก์ไปหวยอื่น) เพิ่มใน sitemap ครบทุกหวยแล้ว
- **โฆษณา:** ตั้ง Variable `ADSENSE_CLIENT=ca-pub-xxxx` แล้วช่องโฆษณาจะขึ้นอัตโนมัติทั้งหน้าแรกและหน้า SEO (ถ้าไม่ตั้ง = ไม่มีโฆษณา)
- **Analytics:** นับยอดเข้าชมหน้า + เลขที่คนตรวจบ่อย (เก็บเป็นตัวเลขรวม ไม่เก็บข้อมูลส่วนตัว) ดูได้ที่หน้า `/admin` หัวข้อ "สถิติการใช้งานเว็บ"
- **สถิติทุกหวย:** หน้า `/stats` เลือกประเภทหวยได้ แสดงสถิติตามโครงสร้าง (รัฐบาล = เลขท้าย 2/3 ตัว + เลขไม่เคยออก, หวยอื่น = 2 ตัวบน/ล่าง, 3 ตัวบน)
- หลัง deploy: ส่ง `/sitemap.xml` ใน Google Search Console เพื่อให้ Google เก็บหน้า landing + หน้าเลขทั้งหมด

## Auto-update หวยอื่น (ลาว/ฮานอย/หุ้น) ผ่าน Provider

หวยกลุ่มนี้ไม่มี API ฟรี — ใช้ระบบ provider ที่เสียบ API สำเร็จรูปได้ (`providers.js`)

**ลองด้วย mock ก่อน:** ตั้ง `PROVIDER=mock` → ระบบจะใส่ผลตัวอย่าง (ฮานอย/ลาว/หุ้นไทยบ่าย) ให้เห็นว่าท่อทำงาน หรือกดปุ่ม "ดึงหวยอื่นจาก Provider" ในหน้า `/admin`

**ต่อ API จริง:** เมื่อสมัครผู้ให้บริการ (เช่น apilotto.com) แล้ว ตั้ง:
- `PROVIDER=http`, `PROVIDER_URL`, `PROVIDER_KEY`
- `PROVIDER_CODE_MAP` = JSON map ชื่อหวยของผู้ให้บริการ → code ของเรา
- ตั้ง `PROVIDER_MINUTES` (ดีฟอลต์ 10) ความถี่ในการดึง

ตัวแปลงใน `httpProvider()` รองรับฟิลด์ยอดนิยม (`3up/top3`, `2down/bottom2`) อยู่แล้ว ถ้ารูปแบบของผู้ให้บริการต่างออกไป ส่ง doc มาให้ผมปรับ `providers.js` จุดเดียวจบ

## อ่านผลจากกลุ่ม Telegram อัตโนมัติ (ฟรี)

ถ้าผลหวยถูกโพสต์ในกลุ่ม/แชนเนล Telegram ที่คุณเข้าถึงได้ บอทอ่านแล้วแกะเลขบันทึกเองได้ ไม่ต้องจ่าย API

**ตั้งค่า:**
1. ตั้ง `TELEGRAM_BOT_TOKEN` (จาก @BotFather)
2. ใน @BotFather: `/setprivacy` → เลือกบอท → **Disable** (ให้บอทเห็นข้อความทั่วไปในกลุ่ม) — ถ้าเป็นแชนเนลต้องตั้งบอทเป็นแอดมิน
3. เอาบอทเข้ากลุ่มที่โพสต์ผล
4. ตั้ง `TELEGRAM_INGEST=1`
5. (แนะนำ) ตั้ง `TELEGRAM_SOURCE_CHATS` = chat id ของกลุ่มนั้น เพื่อรับเฉพาะกลุ่มที่ต้องการ — ดู chat id ได้จาก log `[ingest] ข้ามข้อความจาก chat <id>` ตอนมีข้อความเข้า

ตัวแกะ (`parse.js`) จับ "ประเภทหวย + 3 ตัวบน + 2 ตัวล่าง" จากข้อความหลายรูปแบบ (3 ตัวบน/บน/3up, 2 ตัวล่าง/ล่าง/2down) ถ้ากลุ่มของคุณใช้รูปแบบเฉพาะ ส่งตัวอย่างข้อความมาให้ปรับ regex ได้
